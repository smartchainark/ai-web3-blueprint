> 看到技术博主分享炫酷工作流？10 分钟内复刻出来。

**文档编号**: 018
**日期**: 2025-12-31
**标签**: `Grok` `Notion` `Claude Code` `技能复刻` `访谈协作`

## 概述

本文档介绍如何从 X（Twitter）平台发现有价值的技术分享内容，快速提取核心信息，并通过 Claude Code 自动实现为可复用的技能或工具。

**核心流程**：
```
发现内容 → 提取完整信息 → 结构化存储 → AI 实现
```

**关键工具链**：
- **Grok**：X 平台原生 AI，擅长处理推文/视频内容
- **Notion**：结构化存储，Claude 可直接读取
- **Claude Code**：根据文档自动实现

## 第一部分：复刻流程

### 场景一：视频内容

适用于：博主发布的演示视频、教程视频、产品 Demo

#### Step 1: 获取视频链接

1. 在 X 平台找到目标视频推文
2. 点击分享 → 复制链接
3. 链接格式：`https://x.com/username/status/xxxxx`

#### Step 2: Grok 转文字稿

打开 Grok（X 平台内置或 grok.x.ai），发送：

```
请帮我整理这个视频的完整文字稿，包括：
1. 逐字逐句的解说内容
2. 屏幕上出现的关键文字/代码
3. 操作步骤的描述

视频链接：[粘贴链接]
```

**Grok 优势**：
- 原生支持 X 平台视频
- 无需下载，直接处理
- 能识别屏幕内容

#### Step 3: 存入 Notion

1. 在 Notion 创建新页面
2. 标题：`[博主名] - [内容主题]`
3. 粘贴 Grok 整理的文字稿
4. 补充你的备注（想实现的功能、改进想法等）

#### Step 4: Claude Code 实现

在 Claude Code 终端中：

```
帮我看一下这个 Notion 页面：[Notion 链接]

根据里面的内容，帮我实现成一个 Claude Skill。
```

Claude 会：
1. 读取 Notion 内容
2. 分析工作流结构
3. 创建 Skill 目录和文件
4. 实现核心功能

### 场景二：图文推文

适用于：长推文、Thread、带截图的教程

**Grok 整理**：
```
帮我整理这条推文/Thread 的完整内容：[链接]
提取所有文字、截图中的代码/配置
```

在 Notion 中结构化整理：

```markdown
## 原始内容
[推文原文]

## 核心功能
- 功能 1
- 功能 2

## 实现要点
- 技术栈
- 关键配置
- 注意事项

## 我的改进想法
- ...
```

### 场景三：GitHub 项目

适用于：博主分享的开源项目、代码仓库

```
帮我分析这个 GitHub 项目：[链接]

总结：
1. 项目功能
2. 核心实现逻辑
3. 关键文件结构
4. 使用方法
```

然后：
```
根据这个项目的思路，帮我实现一个类似的 Skill/工具。
不需要完全复制，理解核心逻辑后重新实现。
```

## 第二部分：实战案例 - 访谈协作技能

以下是使用本 SOP 复刻的完整案例：**访谈协作写作技能（interview-collab）**

### 背景

在 X 平台看到博主分享访谈写作技巧视频，10 分钟内复刻成可用的 Claude Skill。

### 复刻过程

```
1. 发现：X 平台博主分享访谈写作技巧视频
2. Grok：整理完整文字稿（含触发词、访谈问题、平台适配规则）
3. Notion：存储文字稿 + 补充个人需求
4. Claude Code：读取 Notion → 自动创建 interview-collab Skill
5. 结果：10 分钟内复刻，立即可用
```

### 技能概述

**核心理念**：好故事是聊出来的，不是直接写出来的

**触发词**：`interview`、`访谈`、`采访我`、`帮我写文章`

**工作流程**：
```
用户输入 → 访谈挖掘（3-5轮）→ 素材整理 → 平台选择 → 风格学习 → 联网补充 → 文章生成
```

**两个阶段**：
1. **访谈阶段**（互动）：像专业记者追问冲突、情绪、洞察，把好故事"聊出来"
2. **写作阶段**（自动）：读取风格参考 → 联网补充素材 → 输出完整文章

### 访谈四要素

| 要素 | 定义 | 追问示例 |
|------|------|----------|
| **冲突** | 转折、意外、矛盾 | "有什么让你意外的地方吗？" |
| **情绪** | 感受、态度 | "当时是什么感受？" |
| **洞察** | 独特视角、深层理解 | "你觉得背后隐藏的含义是什么？" |
| **细节** | 场景、数字、对话 | "能具体描述一下当时的场景吗？" |

**结束条件**：核心观点 + 1-2个故事/细节 + 情绪 + 独特视角 + 潜在金句

### 平台适配

| 参数 | 小红书 | 公众号 | 微博 |
|------|--------|--------|------|
| 字数 | 500-800 | 1500-2500 | <140 或 500 |
| 标题 | emoji+疑问/感叹 | 观点/悬念 | 直接 |
| 开头 | 金句/冲突（3秒抓人） | 故事/背景 | 观点 |
| 结尾 | 互动/共鸣 | 升华/呼吁 | 金句 |

### 技能目录结构

```
interview-collab/
├── SKILL.md                      # 主文件
├── references/
│   ├── SOP.md                    # 完整操作流程
│   ├── interview-questions.md    # 问题库
│   ├── platform-styles.md        # 平台风格指南
│   ├── story-extraction.md       # 故事要素提取
│   └── style-samples/            # 用户风格参考文章
└── scripts/
    └── interview_tracker.py      # 状态追踪脚本
```

### 使用示例

```
用户：interview

Claude：你好！我是你的访谈协作助手。
       首先，你今天想聊什么话题？

用户：想聊聊 Manus 被 Meta 收购这件事

Claude：听到这个消息时，你的第一反应是什么？

用户：挺意外的，因为之前还在讨论他们会不会独立上市...

[3-5轮后]

Claude：好的，我已经收集到足够的素材了！
       你希望文章发布在哪个平台？

用户：小红书

Claude：[输出完整文章]
```

### 实战成果

使用本技能创作的文章《Manus被Meta收购，创业者破防了》：
- 10 分钟完成创作
- 发布到小红书
- 接近 1 万阅读量

## 第三部分：最佳实践

### 选择合适的内容

✅ **适合复刻**：
- 工作流/SOP 类（如访谈写作、代码审查流程）
- 自动化脚本（如批量处理、定时任务）
- Prompt 模板（如写作助手、分析框架）

❌ **不适合复刻**：
- 需要特定 API/服务的产品
- 高度定制化的企业内部工具
- 涉及付费服务的核心功能

### 提高成功率

1. **信息完整性**：文字稿越详细，实现越准确
2. **补充上下文**：告诉 Claude Code 你的使用场景和偏好
3. **迭代优化**：第一版不完美很正常，持续改进

### Notion 组织建议

```
复刻项目/
├── 待处理/
│   └── [博主名] - [主题].md
├── 已实现/
│   └── [Skill名] - 来源：[博主].md
└── 参考资料/
    └── 原始文字稿归档
```
